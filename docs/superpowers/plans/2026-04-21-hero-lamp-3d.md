# Hero Lamp & 3D Scene Integration Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Integrate a Lamp background effect and an interactive Spline 3D scene into the Hero section, transforming it into a dark full-viewport hero with a two-column layout (text left, 3D scene right).

**Architecture:** Create isolated primitive components in `components/ui/` (`lamp.tsx`, `splite.tsx`, `spotlight.tsx`), add a `cn` utility in `lib/utils.ts`, then refactor `Hero.tsx` to import and compose these primitives into the new dark two-column layout. The Hero becomes a dark contrast section (slate-950 + cyan) sitting above the rest of the warm cream-themed site.

**Tech Stack:** Next.js 16 App Router, TypeScript, Tailwind CSS v4, Framer Motion (already installed), @splinetool/react-spline, clsx, tailwind-merge

**Why the inline `style` on the conic-gradient divs:** Tailwind v4 does not ship a `bg-gradient-conic` utility or `--tw-gradient-stops` CSS variable wiring for conic gradients, so we use inline styles with hardcoded cyan (`#06b6d4` = `cyan-500`) to avoid a broken background.

import { Suspense, lazy } from 'react'

const Spline = lazy(() => import('@splinetool/react-spline'))

interface SplineSceneProps {
  scene: string
  className?: string
}

export function SplineScene({ scene, className }: SplineSceneProps) {
  return (
    <Suspense
      fallback={
        <div className="w-full h-full flex items-center justify-center">
          <div className="w-8 h-8 border-2 border-cyan-400 border-t-transparent rounded-full animate-spin" />
        </div>
      }
    >
      <Spline scene={scene} className={className} />

  )
}

## Notes

- **Spline scene URL** — `https://prod.spline.design/kZDDjO5HuC9GJUM2/scene.splinecode` is a placeholder demo. Replace with a custom Spline scene URL when ready.
- **Conic gradient** — Tailwind v4 has no `bg-gradient-conic` utility; the lamp beams use inline `style` with hardcoded `#06b6d4` (cyan-500). This is intentional, not a mistake.
- **`animate-spotlight`** — Requires the custom keyframe added in Task 3 Step 2. Without it the spotlight SVG stays at `opacity: 0` and is invisible.
- **Mobile 3D performance** — The Spline scene is lazy-loaded. On low-powered devices it may load slowly; the spinner fallback handles this gracefully. A future improvement could be to hide the 3D panel on very small screens with `hidden sm:block`.
